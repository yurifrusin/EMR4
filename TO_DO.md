### TO DO: Migrate SNOMED CT AU Extraction to Deterministic API

**Objective:**
Eliminate LLM hallucinations and code drift for clinical diagnoses by shifting from probabilistic generation (Gemini) to deterministic retrieval (Terminology API).

**The Problem (Current State):**
Because the SNOMED ontology is too massive and hierarchical to flatten into our Vector/RAG pipeline, Gemini is relying entirely on its pre-trained memory to output the numeric `snomed_ct_au_code`. Since LLMs are probabilistic, this results in "code drift" between runs—the AI occasionally selects different nodes in the hierarchy (e.g., swapping between parent and child concepts like "Melanoma" vs. "Malignant melanoma of skin") or hallucinates the Australian namespace extension digits.

**The Architecture Fix:**
The LLM should only be responsible for reading comprehension, not database memorization. We need to restrict Gemini to extracting the *text term* only, and let the Python backend handle the numeric mapping via a strict API call.

**Execution Steps:**

1. **Update the LLM Prompt Schema (`main.py`):**
* Modify the expected JSON structure in the prompt to remove the demand for the numeric code.
* Gemini should only output the clinical term: `{"clinical_diagnoses": [{"term": "Malignant melanoma"}]}`.


2. **Register for Terminology Server Access:**
* Acquire API credentials for a dedicated clinical terminology server (e.g., the ADHA National Terminology Server FHIR API or a middleware equivalent).


3. **Build the Python Retrieval Function:**
* Write a new helper function in `main.py` (e.g., `get_snomed_code(clinical_term)`) that takes the text string generated by Gemini and executes a standard HTTP GET request to the terminology server to find the exact, static numeric ID.


4. **Update the Backend Orchestrator:**
* Inside the `/api/v1/analyze-consultation` endpoint, loop through Gemini's extracted `clinical_diagnoses`.
* Pass each term through the new retrieval function.
* Append the verified numeric codes back into the final JSON payload before transmitting the data to the Word sidebar.
